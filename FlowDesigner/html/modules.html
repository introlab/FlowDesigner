SpeechInput Modules

The FreeSpeech project is divided into the following modules:

Sound Interface

The sound interface is the part responsible for picking up the audio from the sound card (via OSS) and make it available for the front-end, the data collection recrder, etc.

Front-End

The front-end converts the audio into a "feature set" that will be used by the recognizer.

HMM (Hidden Markov Model)

The HMM part is a set of classes that represents the speech (Anyone's got something better?).

Training

The training uses sound tokens for which the tanscription is known in order to train an HMM model.

Decoding

The decoding part uses a trained HMM and features from the front-end to decode i.e., find what has been said.

Data Collection

Speech recognition is all about one thing: data. We need data.

Speech Server

The speech server will run decoding in real-time and alllow connections from applications. Similar to the X server.

Client Library

The client library will provide the API for the interaction between the applications and the speech server.
